66  /*
67  Asset Metadata structure format
68  One of fSize, fModifiedData or fUniqueID is mandatory for detecting if file is changed
69  on server.
70  fUniqueID and fVersionData is used to create resource stamp (refer HTTPAssetLinkResourceStateUpdater::CreateResourceStamp)
71  */
72  struct AssetMetadata
73  {
74  uint64 fSize; // size of the asset
75  IDTime fModifiedDate; // modified Date for the asset
76  std::string fUniqueID; // UniqueID which can used to determine the asset state(uptodate/modified)
77  std::string fVersionData; // Version of the asset
78  std::string fPath; // OPTIONAL. Path of the asset on the server.
79  // NOTE - Its used for internal use case and will be removed in future.
80  std::string fGuid; // OPTIONAL. Unique identifier of the asset. Should come from the server.
81  std::string fClientData; // OPTIONAL. This is place where client can put their data for custom use.
82  // It can be leveraged my plugin developer for their own purpose.
83  };
